Verifly
An open-source solution for email verification.
Documentation

Installation
To install and run Verifly please follow the steps below.
  1. Clone the repository
    cd desired/target/directory
    git clone https://github.com/Randware/Verifly
  2. Install dependencies
    npm install
  3. Modify the configuration inside the "auth.env" file. For documentation on this step go to the configuration section.
  4. Run the API
    npm start
Configuration
Verifly uses the "auth.env" file for configuration. Below is a table explaining all configuration variables and a couple of examples for different email providers. For more detailed information, head to the Nodemailer documentation.
Variables
Variables Type Description
apiPort integer The port the API will run on.
host string The smtp host (smtp.example.com).
port integer The host port to connect to.
service string The service of your email. Leave blank if you have a host address.
secure boolean Enable or disable the use of SSL (leave blank if you don't know what you're doing).
secureConnection boolean Enforce the use of TSL.
rejectUnauthorized boolean Allows TLS server with self-signed or invalid TLS certificate.
display_name string The display name of the sender.
user string The email (username) of the sender.
password string The password for the given email.
Examples
  • An example for Outlook:
    apiPort=3030
    host="smtp-mail.outlook.com"
    port=587
    service=""
    secure=
    secureConnection=false
    rejectUnauthorized=false
    display_name="Verifly"
    user="example@outlook.com"
    password="password"
  • An example for Mailtrap:
    apiPort=3030
    host="sandbox.smtp.mailtrap.io"
    port=2525
    service=""
    secure=
    secureConnection=false
    rejectUnauthorized=false
    display_name="Verifly"
    user="0d4f******"
    password="c0629*****"
  • An example for Mailersend:
    apiPort=3030
    host="smtp.mailersend.net"
    port=587
    service=""
    secure=
    secureConnection=false
    rejectUnauthorized=false
    display_name="Verifly"
    user="ex_ample@trial-XXX.mlsender.net"
    password="gGHqZR9t*******"
API Reference
Once the API is up and running, this sections explains how to actually use the API.
To request a verification code and send the email, you have to make a POST request to the /verify endpoint. All available request parameters are documented in a table below.
Request parameters
Parameter Type Description
email string Required The recipient's email.
subject string The subject of the email.
html string The html styled text content of the email (use this if possible).
text string The plain text content of the email.
from string The from text of the email: "display_name <your@mail.com>" (not recommended).
cc string The CC for the email.
bcc string The BCC for the email.
verification_code_length integer The length of the verification code.
verification_code string Your own custom verification code, if desired.
verification_code_special_chars boolean Include special characters in verification code or not.
Email placeholders
These placeholders can be used inside the email content string to insert values from this specific email's context.
Placeholder Description
${verification_code} The placeholder for the verification code.
${emailTo} The placeholder for the recipient's email.
${emailFrom} The placeholder for the sender's email.
${display_name} The placeholder for the sender's name.
Examples
const requestBody = {
email: "example@mail.com",
verification_code_length: 10,
html: "<p>Here is your verification code: ${verification_code}</p>"
}

fetch("http://urlToApi.com/verify", {
method: "POST",
headers: {
'Content-Type': 'application/json'
},
body: JSON.stringify(requestBody)
}).then(res => res.json())
.then(data => console.log(data))
.catch(err => console.log(err));
Common issues
This section will cover some common problems or issues you may come across when using Verifly. It will also provide likely solutions.
Using git
If you encounter an error along the lines of "git was not found" when running the "git clone" command, you probably haven't installed git. Follow this guide to do so.
Using npm
If you encounter an error along the lines of "npm was not found" when running the "npm start" command, you probably haven't installed npm or nodejs. Follow this guide to do so.
Other issues
If you encounter any other issues related to Verifly, please feel free to create an issue on our GitHub repository, so we can look into fixing it.
